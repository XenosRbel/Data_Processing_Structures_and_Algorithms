3.1 Общие сведения о ссылочном типе данных

 

Динамические переменные – это переменные, которые порождаются и уничтожаются в процессе выполнения программы, размер значений которых определяется и может изменяться при выполнении программы.

Для работы с динамическими переменными предусмотрен ссылочный тип (указатель). Значением этого типа является ссылка на какой-либо элемент программы. По данной ссылке осуществляется непосредственный доступ к этому элементу. В качестве ссылки используется адрес соответствующего элемента в памяти машины.

Имеется два вида указателей – указатель на элемент некоторого типа и указатель, не связанный с типом (Pointer).

Пример 3.1. Использование указателей при объявлении программных элементов. Символ ^ называется знаком карата.

        Указатели являются автоматическими переменными ссылочного типа. Динамические переменные не объявляются в программе. Поэтому единственным средством доступа к ним являются указатели. Они занимают четыре байта памяти. Иногда необходимо в качестве значения указателя принять пустую ссылку, которая не связывает с данным указателем никакой динамической переменной. Для этого служит предопределенная константа Nil, которая является совместимой по типу с любым типом указателя.

3.2.2 Процедуры Getmem и Freemem

 

        Использование процедур Getmem и Freemem – это второй метод работы с динамическими переменными.

        Вызов процедуры Getmem имеет следующий формат:

 

        Getmem(P, Size);

 

        Здесь Р – указатель любого типа, Size – выражение типа Word. Процедура Getmem выделяет из Heap-области динамической памяти блок памяти заданного размера Size и адрес этого блока присваивает ссылочной переменной Р. Максимальный размер блока – 65521 байт (64К байта- $F(смещение)).

        Для освобождения блока памяти, занимаемого динамической переменной, используется процедура Freemem:

 

Freemem(P,Size);

 

        Здесь Р – указатель произвольного типа, которому предварительно было присвоено значение процедурой Getmem или оператором присваивания. Значение Size должно точно соответствовать размеру переменной, размещенной до этого процедурой Getmem. Процедура Freemem уничтожает динамическую переменную P^, занимаемая ею память считается свободной, значение Р становится неопределенным.

        Таким образом, действие процедур Getmem/Freemem аналогично действию процедур New/Dispose – порождается и уничтожается динамическая переменная.

3.2.3 Процедуры Mark и Release

 

         Процедуры  Mark (пометить) и Release (освободить) позволяют повысить эффективность работы с динамической памятью. Обращение к процедуре Mark  имеет следующий вид:

         Mark(P);

 

Здесь P – переменная типа указатель, ссылающаяся на переменную любого типа. Процедура Mark присваивает своему параметру P адрес начала свободной области динамической памяти.

         Помеченная область с помощью процедур New или Getmem используется для размещения отдельных динамических переменных.

         Когда динамические переменные окажутся ненужными, занимаемую ими память можно освободить. Для этого используется процедура Release с тем же параметром, что и в предыдущем обращении к процедуре Mark:

        

         Release(P);

 

Процедура  Release  освобождает память, начиная с адреса, полученного в результате выполнения последней процедуры Mark. Значение указателя P после выполнения процедуры Release равно Nil.

         Схематически представление работы с динамической памятью без и с использованием процедур Mark/Release приведено ниже.

 

 

        Параметр процедуры Mark нельзя использовать в качестве параметра процедур New, Getmem, а также его нельзя изменять как минимум до его использования в соответствующей процедуре Release.

        Возможна вложенность процедур Mark.Тогда каждой процедуре Mark соответствует своя процедура Release.

        В одной программе не рекомендуется использовать одновременно процедуры Dispose или Freemem и Mark/Release, так как это может привести к непредсказуемым результатам, поскольку механизмы управления свободной памятью для всех процедур существенно различаются.

        В общем случае вместо любой автоматической переменной в программе можно использовать динамическую переменную.

        Основное достоинство использования динамических переменных – экономия памяти компьютера.

        Недостатки использования динамических переменных по сравнению с автоматическими:

        - удлиняется текст программы за счет использования процедур New/Dispose, Getmem/Freemem, Mark/Release;

        - снижается наглядность программы за счет использования переменной с указателем;

        - снижается быстродействие программы за счет необходимости во время ее выполнения размещать порожденную динамическую переменную в памяти машины, за счет формирования значения соответствующего указателя и усложнения доступа к значению динамической переменной.
